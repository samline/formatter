This section documents the complete public surface of `@samline/formatter`. Pages are grouped by concept (configuration, API, examples, regex) rather than by internal module — readers care about *what* they can do, not how the codebase is organised.

:::note
If you add a new page under `src/content/docs/reference/`, it appears under **Reference** in the sidebar automatically (no edit to `site.config.mjs` required).
:::

## Sections in this reference

- [Configuration](/formatter/reference/configuration/) — every option explained, with defaults and per-`formatType` behavior.
- [API](/formatter/reference/api/) — function-by-function signatures, exported types, and the browser global surface.
- [Examples](/formatter/reference/examples/) — end-to-end recipes for common form scenarios.
- [Regex](/formatter/reference/regex/) — bundled validation patterns paired with ready-to-use error messages.
